1. Tại sao định danh (Naming) quan trọng?

👉 Trong hệ phân tán:

Tài nguyên nằm rải rác nhiều nơi
Muốn truy cập → phải biết tên

📌 Ví dụ:

Bạn gọi: www.google.com
Nhưng máy thật cần: 142.250.xxx.xxx

👉 Naming giúp:

Người dùng dùng tên dễ nhớ
Hệ thống tự map → địa chỉ thật
2. Identifier vs Address
Khái niệm	Ý nghĩa
Identifier	Tên định danh duy nhất (VD: username, domain)
Address	Địa chỉ truy cập (VD: IP, port)

📌 Ví dụ:

Identifier: www.facebook.com
Address: 157.240.xxx.xxx

👉 Khác nhau:

Identifier: ổn định
Address: có thể thay đổi
3. Name Resolution là gì?

👉 Là quá trình:

Tên → Địa chỉ

📌 Ví dụ:

www.example.com → 192.168.1.1
Các cách:
Recursive (đệ quy)
Iterative (lặp)

👉 DNS dùng cả 2

4. Thuộc tính hệ thống naming tốt
Scalability (mở rộng tốt)
Fault tolerance (chịu lỗi)
Efficient (nhanh)
Consistency (đồng nhất)
Transparency (ẩn phức tạp)
